# ADR-0001：采用自包含模块化单体

**状态：** Accepted  
**日期：** 2026-07-19

## Context

现有仓库只有 Vue 原型，旧后端不可验证。项目需要由一个开发者维护、无需云账号即可演示，同时保留清晰的领域边界和真实数据事务。

## Decision

在当前仓库中增加 Fastify 模块化单体和 SQLite 数据库：开发时 Vite 代理，生产时同源托管。数据库使用 WAL、外键和参数化语句；服务端按身份、课程、预约、作业、课堂、管理模块拆分。

## Consequences

- 正面：一条命令可运行；核心后端代码完全在仓库；跨角色事务易验证；部署成本低。
- 负面：SQLite 不适合大量并发写入或多实例横向扩容；达到该规模时需要迁移 PostgreSQL。
- 约束：仓储访问保持集中，不在视图/路由中散落 SQL；外部能力必须通过 provider 接口隔离。
